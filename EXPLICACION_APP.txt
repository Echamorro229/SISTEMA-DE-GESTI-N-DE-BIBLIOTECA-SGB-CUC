SISTEMA DE GESTIÓN DE BIBLIOTECA (SGB) - CUC
==============================================

VISIÓN GENERAL
==============

Es una aplicación web full-stack para la gestión integral de la biblioteca de la Universidad de la Costa, permitiendo que estudiantes, bibliotecarios y administradores gestionen libros, préstamos, reservas y reportes de forma eficiente.


ARQUITECTURA TÉCNICA
====================

FRONTEND (Next.js)
- Interfaz moderna y responsive
- Ubicado en la raíz del proyecto
- Se ejecuta en http://localhost:3000
- Comunicación con backend mediante API REST

BACKEND (NestJS)
- API RESTful robusta
- Ubicado en backend/
- Se ejecuta en http://localhost:4000/api
- Manejo de lógica de negocio y validaciones

BASE DE DATOS (Supabase/PostgreSQL)
- Almacenamiento de todos los datos
- Arquitectura relacional
- Scripts SQL disponibles en backend/supabase/schema.sql


SISTEMA DE AUTENTICACIÓN Y ROLES
=================================

La aplicación implementa control de acceso basado en roles (RBAC):

ROLES DISPONIBLES:

1. ESTUDIANTE - Acceso limitado
   - Puede buscar libros en el catálogo
   - Realizar préstamos y reservas
   - Ver solo sus propias reservas

2. BIBLIOTECARIO - Acceso moderado
   - Gestionar catálogo completo
   - Ver y procesar todos los préstamos y reservas

3. ADMINISTRADOR - Acceso total
   - Gestionar usuarios y roles
   - Reportes y métricas del sistema
   - Configuración completa


MÓDULOS PRINCIPALES DEL BACKEND
================================

1. AUTH MODULE (Autenticación)
   - Login con validación de credenciales
   - Recuperación de contraseña
   - Autenticación con Supabase

2. BOOKS MODULE (Libros)
   - CRUD completo de libros
   - Gestión de copias disponibles
   - Información: título, autor, ISBN, categoría
   
   ENDPOINTS:
   - GET /api/books - Listar todos
   - POST /api/books - Crear nuevo
   - PATCH /api/books/:id - Actualizar
   - DELETE /api/books/:id - Eliminar

3. USERS MODULE (Usuarios)
   - Gestión de usuarios del sistema
   - Asignación de roles
   - Información: nombre, email, rol

4. ROLES MODULE (Roles)
   - Crear y gestionar roles
   - Asignación de permisos

5. LOANS MODULE (Préstamos)
   - Registrar préstamos de libros
   - Fechas de vencimiento
   - Seguimiento de devoluciones
   - Estados: Activo, Vencido, Devuelto
   
   ENDPOINTS:
   - GET /api/loans - Listar préstamos
   - POST /api/loans - Crear préstamo
   - PATCH /api/loans/:id/return - Registrar devolución

6. RESERVATIONS MODULE (Reservas)
   - Sistema de reservas con cola de espera
   - Posición en la fila de espera
   - Estados: Pendiente, En espera, Lista, Confirmada, Cancelada
   
   ENDPOINTS:
   - GET /api/reservations - Listar reservas
   - POST /api/reservations - Crear reserva
   - PATCH /api/reservations/:id/confirm - Confirmar reserva

7. DASHBOARD MODULE (Dashboard)
   - Métricas del sistema
   - Actividad reciente
   - Estadísticas generales

INSTALACIÓN Y EJECUCIÓN
=======================

REQUISITOS:
- Windows 10/11
- Node.js LTS instalado
- npm incluido con Node.js

INSTALACIÓN DE NODE.JS:
Desde PowerShell ejecuta:
winget install OpenJS.NodeJS.LTS --accept-package-agreements --accept-source-agreements

VERIFICAR INSTALACIÓN:
node --version
npm.cmd --version

INSTALACIÓN DEL PROYECTO:

PRIMER PASO - Setup general:
npm.cmd install

SEGUNDO PASO - Frontend:
npm.cmd run dev
Accede a: http://localhost:3000

TERCER PASO - Backend:
cd backend
npm.cmd install
Copy-Item .env.example .env
npm.cmd run start:dev
API disponible en: http://localhost:4000/api

CONFIGURACIÓN SUPABASE:
1. Crear proyecto en Supabase
2. Ejecutar backend/supabase/schema.sql
3. Completar variables de entorno en backend/.env
4. En la raíz, crear .env.local con:
   NEXT_PUBLIC_API_URL=http://localhost:4000/api

COMANDOS ÚTILES:
npm.cmd run build - Compilar proyecto
npm.cmd run lint - Verificar código
npm.cmd run test:e2e - Ejecutar pruebas
npm.cmd audit - Auditar dependencias


TIPOS DE DATOS PRINCIPALES
===========================

BOOK (Libro)
- id: Identificador único
- title: Título del libro
- author: Autor
- isbn: Número ISBN
- category: Categoría
- copies: Número de copias disponibles

LOAN (Préstamo)
- id: Identificador único
- user: Usuario que realiza el préstamo
- book: Libro prestado
- due_date: Fecha de vencimiento
- state: Estado (Activo, Vencido, Devuelto)

RESERVATION (Reserva)
- id: Identificador único
- user: Usuario que realiza la reserva
- book: Libro reservado
- position: Posición en la cola
- state: Estado (Pendiente, En espera, Lista, Confirmada, Cancelada)

USER (Usuario)
- id: Identificador único
- name: Nombre completo
- email: Correo electrónico
- role: Rol del usuario

ROLE (Rol)
- id: Identificador único
- name: Nombre del rol

PUNTOS TÉCNICOS IMPORTANTES
============================

- La aplicación usa fetch API para las llamadas HTTP
- El estado se maneja con React Hooks en el frontend
- El backend valida todas las operaciones
- La base de datos es relacional con Supabase
- Las fechas de vencimiento se calculan automáticamente
- Los permisos se validan en cada acción crítica
- El sistema soporta múltiples sesiones concurrentes
- Las transacciones se manejan a nivel de base de datos
